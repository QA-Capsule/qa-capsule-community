# K6 (load testing)

K6 does **not** emit JUnit by default — use the **JSON webhook** for threshold breaches.

## JSON ingest

```bash
k6 run script.js || true

curl -X POST "${QA_CAPSULE_URL}/api/webhooks/" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ${QA_CAPSULE_API_KEY}" \
  -H "X-Run-Id: ${CI_PIPELINE_ID}" \
  -d '{
    "name": "K6 load test — checkout API",
    "status": "CRITICAL",
    "error": "http_req_duration p(95) exceeded 500ms",
    "framework": "k6"
  }'
```

## JUnit via extension

Community tools can convert K6 summary to JUnit; if you produce `k6-junit.xml`:

```bash
curl -f -S ... "?framework=K6" -F "file=@k6-junit.xml"
```

## Gatling {#gatling}

Same pattern — JSON webhook with `framework: gatling` or export simulation logs and map failures manually.
