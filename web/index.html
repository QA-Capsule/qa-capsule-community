Help

                    
Trigger — Entry point when a CI alert is ingested into QA Capsule.

                    
Condition — Branches on tags (e.g. [FLAKY]), status, or text match. Connect the true (top) or false (bottom) output.

                    
Action — Runs a native integration from the Plugin Engine registry (Slack, Jira, …). Configure gateway routing values first.

                    
Example: Trigger → Condition [FLAKY] → true → Slack; false → Jira.

                    
Enable workflow — When checked and saved, the DAG drives remediation. When unchecked, the graph is stored as a draft and legacy AUTO-RUN + trigger_on rules still apply.

                    
Reset deletes the DAG entirely and restores legacy mode.
