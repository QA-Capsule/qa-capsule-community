Workflow quick guide

                    
Trigger — One entry node per graph when a CI alert is ingested.

                    
Condition — Branch on [FLAKY], [PERF], status, or error text. Top output = true, bottom = false.

                    
Action — Native plugin (Slack, Jira, K8s…). Must be in gateway routing matrix + routing-enabled in Plugin Engine.

                    
Example — Trigger → [FLAKY]? → Slack (yes) / Jira (no).

                    
Enable workflow + Save = DAG live. Unchecked = draft (legacy AUTO-RUN still runs).

                    
Simulate — Dry-run without calling vendors. Reset — delete DAG.
